# Archive

This directory stores project history and process evidence that should not be
treated as current design or implementation input.

Rules:
- `docs/` contains active design and engineering artifacts.
- `archive/` contains historical, review, audit, superseded, or process
  evidence artifacts.
- Agents may read archived files for traceability, but downstream design,
  modeling, planning, and implementation should use active artifacts unless the
  user explicitly asks for historical review.

Current archive sections:
- `reviews/`: review and audit process evidence from design gates and handoffs.
