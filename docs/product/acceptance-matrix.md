# Acceptance Matrix

Required standard:

- `standards/acceptance-matrix-standard.md`

This matrix is the retained product acceptance baseline for the restarted
architecture flow. P0/P1 items cannot be downgraded, deleted, merged away,
weakened, or marked Done without implementation, QA evidence, integration
evidence, audit pass, and no open P0/P1 blocker.

## Scenario-Level Coverage Notes

- ACC-003 must be tested with create, view, edit, search, filter, assign/transfer, missing fields, and unauthorized access.
- ACC-012 must be tested separately for activity, note, and task creation against at least lead, customer, opportunity, contract, and payment contexts.
- ACC-015 must be tested with happy path, empty state, invalid filter, permission-hidden records, and permission-denied detail access.
- ACC-014 is record-local business history. ACC-022 is admin/global operation-log query.

## Notes

- `Done` requires implementation, QA evidence, integration evidence, audit pass, and no open P0/P1 blocker.
- Open P0/P1 blockers prevent release.
- Implementation is blocked until new architecture, MDA, test model, and G8 task planning pass the required Gates again.
