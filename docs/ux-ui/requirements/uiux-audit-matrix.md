# UI/UX 多维审计验收矩阵（uiux-audit-matrix.md）

> 状态：release-owner 确认 2026-06-12。这是本轮系统性 UI/UX 审计的**唯一尺子**。
> 它**操作化**现有 `uiux-implementation.requirements.md` 的 A1–A9 验收项 + C1–C6 约束，
> 把"看着对"升级为"可测、有通过线、有验证方法"。不另起炉灶、不弱化任何 P0/P1。
> 缘起：前几轮审计过度押在视觉保真（D1）单维，功能/导航/响应式/交互/a11y（D2–D6）漏检，
> release owner 逐张截图才暴露——故确立多维 + `isVisible ≠ 可用` 纪律。

## 绑定约束（C1–C6，审计期间不变，红线）

- **C1 仅前端/设计**：不改后端、API、数据模型、业务逻辑、服务边界。
- **C2 无降级**：不弱化任何 P0/P1 功能验收或既往 G12 安全修复（IDOR、durable audit、乐观并发等）。
- **C3 保 zh-CN**：不回退英文；**不得改枚举/角色比较值**（仅经 labels.ts 显示映射）。
- **C4 真实枚举**：只用真值；显示走 labels.ts。
- **C5 e2e 保绿**：现标准 = `workers:2 + retries:1` 下 0-failed、0-skip、断言不减。
- **C6 与锁定 mockup + design-system 一致；科技感来自组件/交互，不靠改色。**

## 审计维度矩阵（D1–D12）

| D | 维度（映射 A 项） | 通过线（Completion Standard） | 验证方法 | 优先级基线 |
|---|---|---|---|---|
| D1 | 视觉保真（A1/A2/C6） | 每页结构/组件/token 与锁定 mockup 一致；无新色 | 1440px 整屏截图 vs mockup + token grep | P1 |
| D2 | **功能完整性**（A2 强化） | 每个"看着能交互"的控件都真生效；无死/静态 UI；无 aria-hidden 假按钮、无硬编码 disabled（除注明的后端限制） | 逐控件实测点击 + grep 死控件模式 | **P0** |
| D3 | **导航/IA**（A2 强化） | 列表↔详情可达可返回；名字/行可点跳；深链生效；无死路 | 实测点击跳转 + 返回 | **P0** |
| D4 | 响应式不破版（A6） | 1024→超宽全程无溢出/裁切/重叠/竖排；≤1024 优雅降级 | ≥3 宽度(1280/1366/1440)整屏截图 + 程序化测 scroll/client 溢出 | P1 |
| D5 | 交互与状态（A3） | loading/空/错误/禁用/成功 + hover/focus/active/selected 全实现；每个动作有可见反馈 | 触发各状态实测 + 截图 | P1 |
| D6 | 可访问性（A5） | 键盘可达可操作、可见 focus 环、焦点管理正确、对比 ≥AA、reduced-motion snap、ARIA 角色/标签 | 键盘 Tab/Enter 实测 + 焦点落点 + elementFromPoint 验最上层 | P1 |
| D7 | 一致性 | 同类控件各页行为/外观/文案一致（如 7 列表行操作模式统一） | 跨页对比 | P2 |
| D8 | 内容/文案（A8/C3） | 全 zh-CN、无原始 key、标签对齐业务术语 | grep 英文 key + 逐页扫 | P1 |
| D9 | 数据正确与边界（A9） | 空/超长/海量值不破版、截断有度、金额/日期格式、规模下分页 | 边界数据实测 + 截断检查 | P1 |
| D10 | 错误处理与韧性 | 失败优雅呈现、可重试、无静默失败 | 触发失败路径 | P2 |
| D11 | 性能/感知 | 无卡顿、动效顺滑、反馈快、骨架屏 | 实测观察 | P2 |
| D12 | 安全/权限 UI（A4/C2） | 角色门控正确、不扩大数据暴露、无越权动作 | 三角色分别实测 | **P0** |

## 审计范围

- 14 个导航页：工作台、线索、公司/客户、联系人、商机、报价、合同、回款、任务、提醒中心、报表、导入/导出、用户与角色、操作日志。
- 工作台 Card→Focus 各 stage 视图（漏斗/阶段/趋势/业绩榜/待办/回款/重点商机/活动）。
- 三角色：Administrator / Sales Manager / Sales（D12 必跑三角色）。

## 审计纪律（强制）

- **`isVisible ≠ 可用`**：交互类用 `elementFromPoint` 验菜单/弹层是该位置最上层且能点；
- **布局类用多宽度整屏截图**（≥3 宽度）由 Claude 人眼通览 + 程序化溢出量化；
- **功能类逐控件实测点击**（不是"DOM 存在"）；
- 发现不靠假设，每条带证据；不在"看着像过"时签字。

## 产出

每个发现：`页面 + 维度(D#) + 现状 + 期望 + 优先级 + 证据`，汇总成**带优先级的缺口清单**（delivery 文档），
P0/P1 进 blockers 踢回 Codex，P2/P3 由 release owner 定批次。Codex 修，Claude 按本矩阵复审。
