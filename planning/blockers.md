# Blockers — CRM System

This register tracks open blockers and pre-gate decisions surfaced during planning.
It is a companion to `planning/gate-status.md` (the gate sync point). On a kickback,
set the affected gate to `Gate Blocked` in gate-status, record it here, and return
to Claude. See `../../../company/collaboration-model.md`.

Status values: `Open` / `In Review` / `Resolved` / `Formal Scope Change by User`.
No-downgrade rule applies: a blocker touching a P0/P1 acceptance may only move to
`Resolved` (with the deciding source recorded) or `Formal Scope Change by User`.

## UI/UX Post-G12 Spot-Fix #6 — Dashboard responsive break at narrower desktop widths (A6) — RESOLVED (release-owner review, 2026-06-10)

Release owner review: at full-screen the dashboard is mostly OK but some text
clips/overflows; at a **non-fullscreen desktop window (~1200–1400px)** the dashboard
**deforms badly** — donut legend labels wrap **character-by-character** (新/商/机
stacked vertically), KPI value `¥12,248,345` overlaps its icon, cards are crushed.
Root cause: the dashboard card grid has **no responsive column reduction** — the
`@media (max-width:1199px)` block collapses the sidebar + other grids but **never
touches `.managerDashboardGrid`/`.salesDashboardGrid`**, which stay
